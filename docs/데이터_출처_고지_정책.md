# 데이터 출처 · 고지 정책

> 기획서 2장 설계 원칙 직결. **타협 불가(🟡).**
> 표심의 생존은 기술이 아니라 신뢰성·중립성에 달려 있다.

## 1. 1차 출처 의무화

모든 표시 데이터는 **1차 출처 링크**를 반드시 동반한다.

| 데이터 종류 | 1차 출처                                    |
| ----------- | ------------------------------------------- |
| 표결        | 국회 회의록 / 열린국회정보 API              |
| 법안        | 의안정보시스템 (likms.assembly.go.kr)       |
| 입법예고    | 국회 입법예고 (pal.assembly.go.kr)          |
| 전과        | 선관위 공보 (info.nec.go.kr)                |
| 선거구·자금 | 공공데이터포털 (data.go.kr)                 |

→ 모든 데이터 테이블은 `source_url`, `last_verified`(수집/검증일) 필드를 일관 적용한다.
→ 관계 그래프 `Relationship.evidence_url`은 **NOT NULL** (엣지마다 근거 필수).

## 2. 데이터 라이선스 / 이용 조건

- 공공데이터(API/포털)는 각 제공처의 **이용약관·라이선스(주로 공공누리)** 를 준수한다.
- 크롤링(의안정보시스템)은 `robots.txt`·서비스 약관·rate limit을 준수하고, 캐싱은 출처 표기를 유지한다.
- 외부 표기 시 "출처: 열린국회정보 / 중앙선거관리위원회 ..." 명시.

## 3. 사실만, 가치판단 배제

- 전과: 죄명·형량·확정여부·선고일·출처만. 해석·논평 금지.
- 표결: 찬/반/기권/불참 사실만.
- 동일 양식 표기로 특정인 부각 금지.
