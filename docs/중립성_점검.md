## 1. 기획서 11장 — 중립성 체크리스트

### 1-1. 문항: 양쪽 논리 대칭, "모름" 항상 제공, 법안명·정당명 질문에 미포함 — ⚠️

| 세부 | 상태 | 근거 |
| --- | --- | --- |
| 양쪽 논리 대칭(장점+단점 둘 다) | ✅ | [seed_questions.py](../backend/app/seed_questions.py) `Draft` 가 `a_pro/a_con/b_pro/b_con` 4필드 강제 — 8문항 전부 양쪽 장점·단점 대칭. UI도 "좋은 점/아쉬운 점" 양쪽 동일 렌더([test/page.tsx](../frontend/app/test/page.tsx) `ChoiceCard`) |
| "모름" 항상 제공 | ✅ | `AnswerChoice.모름`([models.py](../backend/app/models.py)), 모든 문항 화면에 "잘 모르겠어요" 고정 버튼([test/page.tsx](../frontend/app/test/page.tsx)). 채점 시 분모·분자 모두 제외([scoring.py](../backend/app/scoring.py)) |
| 법안명·정당명 질문(body)에 미포함 | ✅ | `body` 는 생활 언어 상황질문, 법안명·정당명은 ▼출처(`source_note`)로만 분리. 자동 검증은 아래 1-1 보강 참고 |
| 어느 쪽도 도덕적 우월 X | ⚠️ | 작성 규칙 반영했으나 **체감은 외부 검수 필요**(특히 8·11번 톤). 로드맵 1-1 잔여 |
| 외부 1~2인 교차검토 | ⛔ | **미완 — 잔여 차단 항목.** 현재 전 문항 `status=초안`, 공개 경로(승인만)에서 0건 노출 |

> **보강(이번 점검에서 추가 권고):** 문항 본문에 정당명·법안명이 섞여 들어가는 회귀를 막기 위해, 시드 단계 자동 가드(정당 약칭/법안 키워드 블랙리스트 검사)를 두면 좋다. 현재는 사람이 작성 규칙을 지키는 방식. → 후속 작업 후보.

### 1-2. 결과: "일치도"로만 표현, "지지/반대해야 한다" 권유 금지 — ✅

- 결과 요약은 "가장 많이 겹친 곳 … %"의 **발견 서술**, 자극 라벨 없음([result/page.tsx](../frontend/app/result/page.tsx)).
- 필수 고지문 `DISCLAIMER` 항상 동봉 — "특정 정당을 지지/반대하라는 뜻이 아니며, 정치적 판단은 본인의 몫"([scoring.py](../backend/app/scoring.py), API 응답에 always 포함 [api.py](../backend/app/api.py)).
- 문항별 비교는 "같은 방향/다른 방향 표결 정당"의 **사실 표기**일 뿐, 권유 문구 없음.

### 1-3. 공유 카드·매칭 결과에도 출처·집계 기준 표기 — ✅

- OG 이미지 하단 고정 문구: "{n}문항 응답 · 실제 국회 표결 일치도 **(지지 반대 권유 아님)**"([api/og/route.tsx](../frontend/app/api/og/route.tsx)).
- 공유 랜딩에도 동일 `disclaimer` 노출([share/page.tsx](../frontend/app/share/page.tsx)).
- 결과 응답에 `method_note`(집계식) 항상 동봉, 화면에 표기([result/page.tsx](../frontend/app/result/page.tsx)).
- 문항별 1차 출처(의안번호 + likms 직링크) ▼접기로 항상 연결.

### 1-4. 의원 카드: 전과·출석률 동일 양식 — ⏳

- Phase 2-4(내 의원 카드)·Phase 3-1(전과 DB) 범위. 현재 MVP에 의원 카드 없음.
- 스키마 차원의 안전장치는 선반영: `CriminalRecord.source_url`, `Person.profile_source_url`/`last_verified`([models.py](../backend/app/models.py)).

### 1-5. 정당 필터 기본값 "전체", 수집 로직 공개 — ✅ (현 범위)

- MVP엔 정당 필터 UI가 없고, 채점은 원내 변별 5당 전체를 항상 집계([scoring.py](../backend/app/scoring.py) `SCORED_PARTIES`) — 특정 정당 부각/제외 없음.
- 집계 로직 공개: `scoring.py` 모듈 docstring + API `method_note`. 정당 입장 출처는 의안번호로 추적 가능.
- 군소·무소속 제외는 "표본 과소 → 일치율 왜곡 방지" 사유를 코드 주석에 명시(중립적 통계 사유).

### 1-6. 자동 생성 문항은 반드시 사람 승인 후 공개 — ✅ (게이트 작동)

- 공개 API 기본값은 `status=승인` 문항만([api.py](../backend/app/api.py) `_public_questions`). 초안은 `?preview=1` 로만(프로토타입).
- 현재 8문항 전부 `초안` → 공개 경로에서 0건(검증: [verify_api.py](../backend/scripts/verify_api.py) "공개=승인만 → 0건" assert).
- 승인 흐름 스키마 준비: `Question.status / created_by / approved_by / approved_at`. LLM 자동 초안 + 어드민 검토 UI는 Phase 2-3.

> ⚠️ **출시 전 확인:** `lib/api.ts` `fetchQuestions(preview=true)` 가 프론트 기본값이다(프로토타입). 실제 공개 시에는 승인 문항 확보 후 `preview=false` 로 전환해야 "승인만 공개" 원칙이 사용자 경로에서도 성립한다. → 로드맵 Phase 1-6 잔여에 반영.

## 3. 개인정보·로깅 (Phase 1-6 신규 — 퍼널 로깅)

이탈 지점 측정을 위해 익명 이벤트 로깅을 추가했다([analytics.ts](../frontend/lib/analytics.ts), [api.py](../backend/app/api.py) `POST /api/events`, [models.py](../backend/app/models.py) `Event`).

| 항목 | 상태 | 근거 |
| --- | --- | --- |
| PII 미수집 | ✅ | 익명 세션ID(localStorage UUID) + 이벤트명 + 소형 props 만. IP·UA 미저장 |
| props 임의 적재 차단 | ✅ | 서버 화이트리스트 이벤트명 + `_sanitize_props`(원시값·키 8개·문자열 120자 상한) |
| 답변 내용 최소화 | ✅ | `answer` 이벤트는 idx·choice(찬/반/모름)만. 자유서술·식별정보 없음 |
| 로깅 실패 격리 | ✅ | sendBeacon 우선 + keepalive fetch 폴백, 실패는 무시(서비스 영향 0) |

> 이벤트 목록(퍼널): `landing → test_start → question_view → answer → test_complete → result_view → share_click`(+ `source_open` = 데이터 신뢰 engagement). "한 단계 더 들어오는가"를 단계별 드롭오프로 측정한다.

## 4. 종합 — 출시 가능 여부

**판정: 조건부 통과.** 중립성 설계 원칙은 코드 레벨에서 대부분 ✅. 출시를 막는 잔여는 *정책/운영*이지 *코드 결함*이 아니다.

**공개 전 반드시 해소해야 하는 잔여(차단):**

1. ⛔ **문항 외부 1~2인 교차검토 → `status=승인` 전환** (1-1) — 사람 필요.
2. ⚠️ **프론트 `fetchQuestions` 기본값 `preview=false` 전환** — 승인 문항 확보 후. 그래야 "승인만 공개"가 사용자 경로에서 성립(1-6).
3. ⏳ **Postgres 기동 + 실 ETL 적재** — 현재 채점은 `CURATED_PARTY_STANCES`(의안번호로 출처 보유한 집계 사실) 폴백. 실 `VoteRecord` 경로는 구현·검증됨.

**권고(차단 아님, 신뢰 강화):**

- 문항 본문 정당명·법안명 자동 가드(시드 단계 검사) 추가.
- 사용자 편향 신고 창구(13장) — Phase 2-3과 함께.

> 이 잔여는 [로드맵.md](로드맵.md) Phase 1-6 "잔여 차단" 항목과 일치한다. 운영자 승인 전까지 공개 경로 노출 0건이 보장된다(verify_api.py assert).
