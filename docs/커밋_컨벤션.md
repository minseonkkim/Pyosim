# 커밋 컨벤션

[Conventional Commits](https://www.conventionalcommits.org/) 기반.

## 형식

```
<type>(<scope>): <subject>

<body>

<footer>
```

- **subject**: 한국어 또는 영어, 50자 이내, 명령형. 끝에 마침표 X.
- **scope**(선택): 변경 영역. 모노레포이므로 권장. 예: `frontend`, `backend`, `etl`, `infra`, `db`, `docs`.

## type 목록

| type       | 의미                                          |
| ---------- | --------------------------------------------- |
| `feat`     | 새 기능                                        |
| `fix`      | 버그 수정                                       |
| `docs`     | 문서                                           |
| `style`    | 포맷·세미콜론 등 동작에 영향 없는 변경          |
| `refactor` | 리팩터링 (기능 변경·버그 수정 아님)            |
| `perf`     | 성능 개선                                       |
| `test`     | 테스트 추가·수정                                |
| `build`    | 빌드 시스템·의존성                              |
| `ci`       | CI 설정                                        |
| `chore`    | 기타 잡무 (위에 안 맞는 것)                     |
| `data`     | 데이터 수집/시드/마이그레이션 관련 (프로젝트 고유) |

## 예시

```
feat(frontend): 정치성향 테스트 진입 화면 구현
fix(etl): 표결기록 수집 시 기권 choice 매핑 오류 수정
data(db): 정당 시드 데이터에 color_hex 추가
docs: Phase 0 셋업 README 작성
```

## 브랜치

- `main` — 항상 배포 가능 상태 유지
- `feat/<설명>`, `fix/<설명>` — 작업 브랜치 → PR → `main`
