## 0. 철학 — "쉽고, 친근하고, 어느 편도 아니다"

표심을 쓰는 사람은 **정치를 잘 모르고, 정치 콘텐츠에 거부감이 있는 사람**이다.
그래서 디자인의 첫 번째 일은 "이거 내 얘기네, 부담 없네"라고 느끼게 하는 것이다.

세 가지 원칙:

1. **접근성 우선 (Approachable)** — 진입 장벽 0. 큰 글씨, 넉넉한 여백, 둥근 모서리,
   한 화면에 한 가지. 회색 정부 문서가 아니라 친구가 보여주는 카드 같아야 한다.
2. **중립성 (Neutral by design)** 🟡 — 색·이미지·레이아웃 어디에도 정당 편향이 없어야 한다.
3. **MZ 감성 (Fresh)** — 2025–26 한국 웹의 톤: Pretendard 굵은 타이포, 강한 흑백 대비,
   넉넉한 라운드, 통통 튀는 마이크로 인터랙션.

### 왜 무채색(잉크)인가 — 브랜드 서사

> 우리는 **색조차 편들지 않는다.** 앱에서 유일하게 채도 있는 색은 '정당 데이터'뿐이다.

한국엔 정당이 너무 많아 거의 모든 색이 어딘가의 정당색이다(네이비=민주, 빨강=국힘·진보,
파랑=조국혁신, 주황=개혁신당·사민, 민트=기본소득, **보라=여성의당** …).
"아무 당도 안 쓰는 색"은 사실상 없다. 그래서 표심은 **브랜드에서 색을 빼버린다.**

브랜드는 거의 검정(Ink) + 흰 + 회색. 컬러는 **오직 정당 데이터**(차트)에서만 등장한다.
그 결과 — 결과 화면에서 색이 켜지는 순간, 그건 "데이터가 말하기 시작했다"는 신호가 된다.
무채색은 비겁한 회피가 아니라 **가장 또렷한 중립 선언**이고, 동시에 굵은 타이포·강한 대비로
밀어붙이면 2025–26 에디토리얼 MZ 톤(뉴닉·토스 계열)으로 가장 신선하다.

## 1. 디자인 토큰

모든 값은 CSS 변수로 정의하고, 컴포넌트는 raw 값 대신 토큰만 참조한다.
구현 블록은 [§6](#6-구현-globalscss-매핑)에 통째로 있다.

### 1.1 색 — 브랜드 (Pyo Ink, 무채색)

브랜드는 단일 무채색 스케일. 인터랙티브(버튼)는 거의 검정, 강조는 대비로만 만든다.
**브랜드에는 채도 있는 색을 절대 쓰지 않는다.**

| 토큰 | HEX | 용도 |
| --- | --- | --- |
| `--ink-950` | `#0E0D12` | 다크 패널 배경, 최강 강조 |
| **`--ink-900`** | **`#18171D`** | **인터랙티브 기본** — CTA 배경, 제목(흰 텍스트 AA ✓) |
| `--ink-800` | `#2A2932` | CTA hover, 다크 표면 |
| `--ink-700` | `#3D3B45` | 본문 텍스트 |
| `--ink-600` | `#56535F` | 본문 보조 |
| `--ink-500` | `#6E6B77` | 보조 텍스트(muted) |
| `--ink-400` | `#9794A0` | 플레이스홀더, 비활성 |
| `--ink-300` | `#C7C5CE` | 구분선(강) |
| `--ink-200` | `#E4E2E9` | 보더 기본 |
| `--ink-100` | `#F0EFF3` | 표면(surface), 선택 배경 |
| `--ink-50` | `#F7F7FA` | 페이지 배경 |
| `--white` | `#FFFFFF` | 카드 표면 |

- **잉크 패널(특별한 순간)**: `linear-gradient(160deg,#2A2932 0%,#0E0D12 100%)` + 흰 텍스트.
  히어로·공유 카드 등에만. 컬러 그라데이션은 쓰지 않는다(무채색 유지).

### 1.2 색 — 시맨틱 (상태, 기능 전용)

UI 상태 신호. **브랜드가 아니라 기능**이며, 최소한으로만 쓴다.
정당 데이터·차트 근처에서는 쓰지 않는다(빨강·주황이 정당으로 읽힐 수 있음).

| 토큰 | HEX | 용도 |
| --- | --- | --- |
| `--success` | `#138A5E` | 완료, 긍정 |
| `--warning` | `#B36B00` | 주의(폼 검증 등) |
| `--danger` | `#C7314F` | 오류 — 국힘 `#E61E2B`·진보 `#D6001C`와 구분되는 마젠타 레드 |
| `--info` | `var(--ink-700)` | 안내 = 무채색(파랑 금지) |

### 1.3 색 — 데이터 시각화 (정당 팔레트) 🟡

**앱에서 유일하게 채도 있는 색.** 정당 데이터 표현 전용이며 브랜드/UI엔 절대 쓰지 않는다.
출처: `backend/app/seed.py`. 차트에서 정당색을 쓸 때 **반드시 텍스트 라벨을 동반**한다
(색만으로 구분 금지 — 색맹 접근성 + 중립성).

| 정당 | HEX | 정당 | HEX |
| --- | --- | --- | --- |
| 더불어민주당 | `#152484` | 진보당 | `#D6001C` |
| 국민의힘 | `#E61E2B` | 기본소득당 | `#00D2C3` |
| 조국혁신당 | `#0073CF` | 사회민주당 | `#F58400` |
| 개혁신당 | `#FF7920` | 무소속 | `#777777` |

규칙:
- 막대·범례·툴팁에 **정당명 텍스트를 항상 함께** 표기.
- 정렬은 일치율·가나다 등 **중립 기준**만. 특정 정당을 시각적으로 우대(맨 위 고정·강조)하지 않는다.
- 막대 두께·여백·라운드는 모든 정당 **동일**(동일 양식 원칙, 기획서 11장).

### 1.4 타이포그래피 — 무채색 브랜드의 주인공

색을 뺀 만큼 **타이포가 브랜드의 인상을 책임진다.** 굵게, 크게, 또렷하게.

**서체: [Pretendard](https://github.com/orioncactus/pretendard) Variable** — 한국어 웹의 사실상 표준,
가독성·중립성·MZ 톤을 동시에 만족. 영문/숫자도 한 패밀리로 일관.
폴백: `system-ui, -apple-system, "Apple SD Gothic Neo", "Malgun Gothic", sans-serif`.

| 토큰/역할 | size / line-height | weight | letter-spacing |
| --- | --- | --- | --- |
| `display` (히어로) | 34 / 42 | 800 | -0.03em |
| `h1` | 27 / 35 | 800 | -0.02em |
| `h2` (문항 본문) | 21 / 30 | 700 | -0.01em |
| `h3` | 18 / 26 | 700 | -0.01em |
| `body-lg` | 17 / 27 | 500 | 0 |
| `body` (기본) | 15.5 / 24 | 450 | 0 |
| `caption` | 13 / 19 | 500 | 0 |
| `micro` (배지) | 11.5 / 16 | 600 | 0.02em |
| `numeral` (큰 %) | 64 / 64 | 800 | -0.02em + `tabular-nums` |

**한글 조판 규칙(필수):**
- `word-break: keep-all;` — 단어 중간 줄바꿈 방지(한글 가독성 핵심).
- `line-break: strict;`
- 제목은 음수 자간(-2~3%)으로 또렷하게, 본문은 자간 0.
- 큰 숫자(일치율 %)는 `font-variant-numeric: tabular-nums;` 로 정렬 안정화.

### 1.5 간격 (Spacing) — 4px 베이스

| 토큰 | px | | 토큰 | px |
| --- | --- | --- | --- | --- |
| `--space-1` | 4 | | `--space-6` | 24 |
| `--space-2` | 8 | | `--space-8` | 32 |
| `--space-3` | 12 | | `--space-10` | 40 |
| `--space-4` | 16 | | `--space-12` | 48 |
| `--space-5` | 20 | | `--space-16` | 64 |

기본 리듬: 카드 내부 패딩 `--space-4`(16), 섹션 간 `--space-6`~`8`, 화면 좌우 `--space-5`(20).

### 1.6 라운드 (Radius) — 넉넉하게(MZ)

| 토큰 | px | 용도 |
| --- | --- | --- |
| `--radius-xs` | 6 | 배지, 작은 태그 |
| `--radius-sm` | 10 | 인풋, 작은 버튼 |
| `--radius-md` | 14 | 카드, 선택지, 기본 버튼 |
| `--radius-lg` | 20 | 큰 카드, 모달, 공유 카드 |
| `--radius-xl` | 28 | 히어로 패널 |
| `--radius-full` | 999 | 진행률 바, pill 버튼, 칩 |

### 1.7 그림자 (Elevation)

은은하게. 무채색 그림자로 통일.

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `--shadow-sm` | `0 1px 2px rgba(14,13,18,.06)` | 보더 보조 |
| `--shadow-md` | `0 4px 16px rgba(14,13,18,.08)` | 카드, 떠 있는 버튼 |
| `--shadow-lg` | `0 12px 32px rgba(14,13,18,.16)` | 모달, 공유 카드 |
| `--ring` | `0 0 0 3px rgba(24,23,29,.22)` | 포커스 링(무채색) |

### 1.8 모션

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `--ease-out` | `cubic-bezier(.2,.8,.2,1)` | 기본(통통 튀는 느낌) |
| `--ease-spring` | `cubic-bezier(.34,1.56,.64,1)` | 선택·등장 강조 |
| `--dur-fast` | `120ms` | hover, 색 변화 |
| `--dur-base` | `200ms` | 진행·전환 |
| `--dur-slow` | `320ms` | 페이지/카드 등장 |

- 마이크로 인터랙션: 선택지 탭 시 살짝 `scale(0.98)`→복귀, 진행률 바 부드럽게 채워짐,
  결과 막대 0%→값으로 그로우.
- **`prefers-reduced-motion: reduce`** 시 모든 트랜지션/애니메이션 제거(접근성 필수).

### 1.9 레이아웃 · 브레이크포인트

- **모바일 우선.** 콘텐츠 최대폭 `640px`, 좌우 패딩 `--space-5`.
- 브레이크포인트: `sm 480` / `md 768` / `lg 1024`. 대부분 단일 컬럼 유지(테스트는 폭 넓힐 이유 없음).
- 터치 타깃 최소 `44×44px`.

### 1.10 다크 모드

`prefers-color-scheme: dark` + `[data-theme="dark"]` 토글 모두 지원. 무채색 브랜드라 반전이 자연스럽다.
배경은 `--ink-950`, 표면 `--ink-900`, 텍스트 `--ink-100`, CTA는 흰색 배경+검정 텍스트로 뒤집는다. (매핑은 §6)

## 2. 컴포넌트

각 컴포넌트는 토큰만 참조. 아래는 스펙 + 적용 위치.

### 2.1 버튼 `Button`

| 변형 | 배경 | 텍스트 | 용도 |
| --- | --- | --- | --- |
| `primary` | `--ink-900` (hover `#000`) | 흰색 | 주요 액션(테스트 시작, 결과 보기, 공유) |
| `ghost` | `--ink-100` / 보더 `--ink-200` | `--ink-900` | 보조(이전, 처음으로) |
| `subtle` | 투명 / 텍스트 `--ink-700`, 밑줄 | — | 3순위, 링크형 |

- 크기: `lg`(높이 52, 본문 16/700) 기본, `md`(44), `sm`(36).
- 모서리 `--radius-md`(일반) 또는 `--radius-full`(pill, 히어로 CTA).
- 상태: hover 색 변화(`--dur-fast`), active `scale(.98)`, focus `--ring`, disabled opacity .4.
- 풀폭(`block`)은 모바일 주요 CTA에 권장.

### 2.2 선택지 카드 `ChoiceCard` (테스트의 핵심)

Ⓐ/Ⓑ 장단점 비교 카드. **이 화면이 서비스의 인상을 결정한다.**

- 기본: `--white` 배경, 보더 `1.5px --ink-200`, 라운드 `--radius-md`, 패딩 `--space-4`.
- hover: 보더 `--ink-400`.
- **선택됨**: 보더 `2px --ink-900`, 배경 `--ink-100`, 좌측 4px 검정 액센트 바(선택 시 등장),
  우상단 체크 ✓. 탭 시 `--ease-spring` 으로 살짝 팝. (색이 아니라 **굵기·대비**로 선택을 표현)
- 라벨(Ⓐ/Ⓑ): `h3` 굵게. 장점 👍 / 단점 👎 행은 `caption`, muted.
  → "좋은 점 / 아쉬운 점"으로 톤 부드럽게(§3).
- "잘 모르겠어요"는 동급 카드지만 중앙 정렬·뉴트럴(부끄럽지 않게, 항상 동등하게 제공).

### 2.3 진행률 `Progress`

- 트랙 `--ink-200`, 채움 `--ink-900`, 높이 6, `--radius-full`.
- 값 변화 `--dur-base --ease-out`. "n / 8" 카운터는 `caption` muted.
- 상단 고정(sticky)으로 항상 보이게 → 이탈 방지("얼마 안 남았네").

### 2.4 카드 / 섹션 `Card`

- `--white` 배경, 보더 `--ink-200`, 라운드 `--radius-md`, 패딩 `--space-4`, `--shadow-sm`.
- 강조 카드(결과 요약): `--ink-100` 배경 또는 **잉크 패널**(다크) + 흰 텍스트로 "특별한 순간" 연출.

### 2.5 배지 · 칩 `Badge` / `Chip`

- 배지: `micro`, 라운드 `--radius-xs`, 무채색 틴트 배경(예: 초안=`--ink-100` + `--ink-600` 텍스트).
- 칩(쟁점 태그 "경제·시장"): `caption`, `--ink-100` 배경, `--ink-700` 텍스트, pill.

### 2.6 정당 일치율 차트 `PartyChart` 🟡

- 수평 막대, 막대색 = 정당색(§1.3 — **앱에서 유일한 컬러**), **막대 옆 정당명 + % 항상 표기**.
- 모든 막대 동일 두께·라운드(`--radius-xs`)·간격. 1위만 살짝 크게 등의 차별 금지.
- 등장 시 0→값 그로우(`--dur-slow`), reduced-motion 시 즉시.
- 축선·배경은 `--ink-200` 이하로 최소화(무채색 UI 위에서 정당 컬러가 또렷이 살아나게).

### 2.7 고지문 `Disclaimer` 🟡 (필수)

- 배경 `--ink-50`, 보더 `--ink-200`, 라운드 `--radius-md`, 패딩 `--space-4`, 텍스트 `caption` muted.
- 아이콘 `⚖️`. 결과·공유 카드에서 **항상 노출**. 디자인적으로 숨기거나 접지 않는다
  ("표결 일치도일 뿐" 메시지는 시스템 신뢰의 핵심).

### 2.8 앱 셸 / 출처 접기 `Source ▼`

- 상단 미니 헤더(로고 "표심" 워드마크, `--ink-900`), 본문 `<main>` 640px.
- 출처 접기: `<details>` 기본, summary `caption` muted "원래 어떤 법안인지 ▼".
  열면 `source_note` + 의안정보시스템 링크. **출처는 항상 한 탭 거리**(신뢰).

## 3. 보이스 & 톤 (UX 라이팅)

디자인의 절반은 글이다. 정치 거부감을 푸는 건 색보다 말투다.

- **친근한 존댓말.** "~해요"체. 반말은 쓰지 않되 딱딱한 "~하십시오"도 금지.
  예: "내 정치성향 알아보기", "거의 다 왔어요!"
- **정치 용어 0.** 법안명·정당명·전문용어는 본문에 절대 노출 금지 → 생활 언어로.
  (출처 ▼ 안에서만 정식 명칭 노출)
- **중립 프레이밍.** "지지/반대하세요" ❌ → "내 생각과 얼마나 닮았는지" ✅.
  결과는 "발견"으로: "나와 가장 많이 겹친 곳은…".
- **부끄럽지 않게.** "잘 모르겠어요"를 당당한 선택지로. "틀렸다/맞다" 표현 금지.
- **자극·낙인 라벨 금지** 🟡. "당신은 ○○주의자" 같은 규정 ❌ → 중립 서술.
- 숫자·결과는 단정적이되 겸손하게: "표결 일치도일 뿐, 판단은 본인 몫".

## 4. 접근성 (Accessibility)

타협 불가. 정치 모르는 사람 + 모든 신체 조건을 포함.

- **색 대비**: 본문 텍스트 ≥ 4.5:1, 큰 텍스트/UI ≥ 3:1 (WCAG AA).
  무채색 브랜드라 대비 확보가 쉽다 — CTA `--ink-900`+흰 텍스트(≈ 15:1), muted `--ink-500`+흰 배경 AA 충족.
- **색만으로 정보 전달 금지** 🟡: 정당·상태는 항상 **텍스트/아이콘/패턴 동반**
  (색맹 + 중립성 이중 이유). 정당색 일부는 서로 가까움(민주 네이비 vs 조국혁신 블루) → 라벨 필수.
- **터치 타깃** ≥ 44×44px, 인접 타깃 간 8px+.
- **포커스 가시화**: 모든 인터랙티브에 `--ring` + `:focus-visible`.
- **모션 축소**: `prefers-reduced-motion` 존중.
- **시맨틱 마크업**: 버튼은 `<button>`, 진행률 `role="progressbar"` + aria, 폼 라벨 연결.
- **한국어 `lang="ko"`**, 스크린리더용 대체텍스트(차트 요약을 텍스트로도 제공).

## 5. 🟡 중립성 가드레일 (디자인 차원)

기획서 11장을 디자인 규칙으로 번역한 체크리스트. PR 리뷰 시 확인.

- [ ] 브랜드/UI에 **채도 있는 색을 쓰지 않았다**(컬러는 정당 데이터에서만).
- [ ] 정당 데이터는 **항상 텍스트 라벨 동반**(색만 의존 금지).
- [ ] 정당 표현 양식이 **전부 동일**(두께·여백·라운드·정렬 기준 중립).
- [ ] 결과·공유 카드에 **고지문이 노출**되어 있다(숨김·접힘 금지).
- [ ] 카피에 정치 용어·정당명·자극/낙인 라벨이 없다.
- [ ] 출처(▼)가 한 탭 거리 안에 있다.
- [ ] 좌/우, 찬/반의 **시각적 무게가 대칭**이다(한쪽을 크게/위에 두지 않음).

## 6. 구현 (globals.css 매핑)

아래를 `frontend/app/globals.css` `:root` 로 교체·확장한다.
(현재 `--accent: #152484` = 민주당색은 **제거**하고 `--ink-*`로 대체.)

```css
:root {
  /* 브랜드 (Pyo Ink — 무채색) */
  --ink-950:#0E0D12; --ink-900:#18171D; --ink-800:#2A2932; --ink-700:#3D3B45;
  --ink-600:#56535F; --ink-500:#6E6B77; --ink-400:#9794A0; --ink-300:#C7C5CE;
  --ink-200:#E4E2E9; --ink-100:#F0EFF3; --ink-50:#F7F7FA; --white:#FFFFFF;
  --ink-panel:linear-gradient(160deg,#2A2932 0%,#0E0D12 100%);

  /* 시맨틱 (기능 전용, 최소 사용) */
  --success:#138A5E; --warning:#B36B00; --danger:#C7314F; --info:#3D3B45;

  /* 역할 별칭(컴포넌트는 이걸 참조) */
  --bg:var(--ink-50); --surface:var(--white); --fg:var(--ink-900); --muted:var(--ink-500);
  --border:var(--ink-200); --primary:var(--ink-900); --primary-strong:#000000;

  /* 간격·라운드 */
  --space-1:4px; --space-2:8px; --space-3:12px; --space-4:16px; --space-5:20px;
  --space-6:24px; --space-8:32px; --space-10:40px; --space-12:48px; --space-16:64px;
  --radius-xs:6px; --radius-sm:10px; --radius-md:14px; --radius-lg:20px;
  --radius-xl:28px; --radius-full:999px;

  /* 그림자·링 */
  --shadow-sm:0 1px 2px rgba(14,13,18,.06);
  --shadow-md:0 4px 16px rgba(14,13,18,.08);
  --shadow-lg:0 12px 32px rgba(14,13,18,.16);
  --ring:0 0 0 3px rgba(24,23,29,.22);

  /* 모션 */
  --ease-out:cubic-bezier(.2,.8,.2,1); --ease-spring:cubic-bezier(.34,1.56,.64,1);
  --dur-fast:120ms; --dur-base:200ms; --dur-slow:320ms;
}

[data-theme="dark"] { 
  --bg:#0E0D12; --surface:#18171D; --fg:#F0EFF3; --muted:#9794A0;
  --border:#2A2932; --primary:#FFFFFF; --primary-strong:#F0EFF3;
}
@media (prefers-color-scheme: dark) { :root:not([data-theme="light"]) {
  --bg:#0E0D12; --surface:#18171D; --fg:#F0EFF3; --muted:#9794A0;
  --border:#2A2932; --primary:#FFFFFF; --primary-strong:#F0EFF3;
} }
/* 다크에서 primary 버튼은 흰 배경 + 검정 텍스트로 뒤집는다(.btn 규칙에서 처리) */

@media (prefers-reduced-motion: reduce) {
  * { animation:none !important; transition:none !important; }
}
```

**Pretendard 적용** (권장: 셀프호스트로 CLS·오프라인 안정):

```bash

```

### 적용 로드맵 (이 문서 → 코드)

1. `globals.css` 토큰 교체(`--accent` 제거 → `--ink-*`) + Pretendard 도입.
2. 기존 유틸 클래스(`.btn`, `.choice`, `.progress`, `.card`, `.disclaimer`)를 신규 토큰으로 리팩터.
   선택/포커스/진행률 등 기존에 `--accent`(보라/민주색)로 표현하던 강조를 **무채색 대비**로 전환.
3. `app/page·test·result·share`, `PartyChart`, OG 라우트 색을 `--accent`→`--ink-*`로 치환.
   (특히 OG 카드 워드마크 `#152484` → `--ink-900 #18171D`. 결과 % 강조색은 **정당색 유지** — 데이터이므로 OK.)
4. 다크모드·reduced-motion·focus-visible 점검 → §5 중립성 체크리스트 통과 확인.
