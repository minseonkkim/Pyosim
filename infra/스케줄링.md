# 자동 갱신 스케줄링 (🤖 1일 1회)

> 감시견 알림(`app/watch.py`)은 **DB의 현재 상태를 실시간 diff** 한다. 즉 DB만 매일
> 새로 채우면 구독자 받은함이 자동 점등된다. 그 '매일 채우기'가 아래 한 줄이다.

```bash
cd etl
python -m jobs.run --job daily --trigger scheduler
```

- 실행되는 잡(순서): `members → bills → proposers → vote_records → bill_stages → petitions → categorize`
  (감시견이 추적하는 청원·법안·의원을 신선하게 유지하는 가벼운 공식-API 잡만).
- **잡별 격리**: 한 잡이 실패해도 나머지는 계속 진행. 부분 실패 시 **종료코드 1**(스케줄러가 실패 감지).
- 실행마다 `pipeline_run` 한 행 기록 → `GET /api/health/pipeline` 로 마지막 실행·성공 시각 확인(모니터링·'데이터 기준일').
- 무거운 작업(스크래핑·LLM·연단위)은 일일 세트에서 제외 — 필요 시 수동/주간으로:
  `bill_content` · `bill_summary` · `lawnotice_opinions` · `propose_dates` · `petition_content` · `budget`.
- 일부 잡만: `--only petitions,bill_stages` / 미리보기: `--dry-run`.

환경변수는 `etl/.env`(`ASSEMBLY_API_KEY` 등) · `DATABASE_URL` 필요.

---

## 1. Cloud Scheduler → Cloud Run Job (운영 권장)

배포 환경(Cloud Run). Cloud Run **Job** 으로 ETL 이미지를 올리고 Cloud Scheduler 가 매일 1회 트리거.

```bash
# 잡 정의(엔트리포인트가 daily 를 실행하도록)
gcloud run jobs create pyosim-daily \
  --image $IMAGE --region $REGION \
  --set-secrets ASSEMBLY_API_KEY=assembly-key:latest,DATABASE_URL=database-url:latest \
  --command python --args="-m,jobs.run,--job,daily,--trigger,scheduler"

# 매일 04:10 KST 트리거 (한국시간 = Asia/Seoul)
gcloud scheduler jobs create http pyosim-daily-trigger \
  --schedule "10 4 * * *" --time-zone "Asia/Seoul" \
  --uri "https://$REGION-run.googleapis.com/apis/run.googleapis.com/v1/namespaces/$PROJECT/jobs/pyosim-daily:run" \
  --http-method POST --oauth-service-account-email $SA_EMAIL
```

> 시각은 국회 공식 소스 갱신이 끝난 새벽으로(예: 04:10 KST). 실패 모니터링은
> Cloud Scheduler 재시도 + `GET /api/health/pipeline` 의 `last_success_at` 으로.

## 2. cron (단일 서버/VM)

```cron
# 매일 04:10 — 표심 자동 갱신
10 4 * * *  cd /srv/pyosim/etl && /srv/pyosim/.venv/bin/python -m jobs.run --job daily --trigger scheduler >> /var/log/pyosim-daily.log 2>&1
```

## 3. Windows 작업 스케줄러 (로컬 개발/단일 PC)

PowerShell 로 매일 04:10 작업 등록(로컬 DB·venv 경로에 맞게 수정):

```powershell
$action  = New-ScheduledTaskAction -Execute "python" `
  -Argument "-m jobs.run --job daily --trigger scheduler" -WorkingDirectory "C:\Pyosim\etl"
$trigger = New-ScheduledTaskTrigger -Daily -At 4:10AM
Register-ScheduledTask -TaskName "Pyosim 자동 갱신" -Action $action -Trigger $trigger
```

확인: `Get-ScheduledTask -TaskName "Pyosim 자동 갱신"` · 즉시 실행 `Start-ScheduledTask -TaskName "Pyosim 자동 갱신"`.

## 모니터링

```bash
curl http://localhost:8000/api/health/pipeline
```

```json
{
  "last_run": { "started_at": "...", "ok": true, "fail_count": 0, "failed_jobs": [], "duration_ms": 123456 },
  "last_success_at": "2026-06-21T04:12:00Z",
  "notice": "데이터는 국회 공식 소스를 1일 1회 자동 갱신합니다. ..."
}
```

- `last_success_at` = 신선도 기준(프론트 '데이터 기준일' 노출용).
- `failed_jobs` 가 비어 있지 않으면 해당 잡만 수동 재실행: `python -m jobs.run --job <name>`.
