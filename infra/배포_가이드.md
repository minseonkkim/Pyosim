# 표심 · Pyosim — GCP 배포 가이드 (Cloud Run)

모노레포 4개 컴포넌트(프론트·백엔드·ETL·DB)를 전부 GCP에 올린다.
스크립트는 [deploy.sh](deploy.sh), 자동 갱신 스케줄은 [스케줄링.md](스케줄링.md) 참조.

## 1. 아키텍처 구조도

```
                              사용자(브라우저)
                                   │ HTTPS
                                   ▼
                        ┌──────────────────────┐
                        │  Cloud Run: pyosim-web │  Next.js 15 (standalone)
                        │  (프론트, 공개)         │  NEXT_PUBLIC_API_BASE = ↓
                        └───────────┬────────────┘
                                    │ fetch (HTTPS, CORS 허용)
                                    ▼
                        ┌──────────────────────┐
                        │  Cloud Run: pyosim-api │  FastAPI (uvicorn)
                        │  (백엔드, 공개)         │
                        └───────────┬────────────┘
                                    │ Cloud SQL 소켓 (/cloudsql/…)
                                    ▼
                        ┌──────────────────────┐
            ┌──────────▶│  Cloud SQL (Postgres) │◀──────────┐
            │ 적재(쓰기) │  pyosim-db             │  읽기      │
            │           └──────────────────────┘           │
            │                                               │
   ┌────────┴──────────┐                          (백엔드가 같은 DB 조회)
   │ Cloud Run Job      │  ETL (backend+etl 동봉)
   │ pyosim-daily       │
   └────────▲──────────┘
            │ :run (OAuth, 매일 04:10 KST)
   ┌────────┴──────────┐
   │ Cloud Scheduler    │  "10 4 * * *" Asia/Seoul
   │ pyosim-daily-trigger│
   └───────────────────┘

   비밀값:  Secret Manager ──(secretAccessor)──▶ api / migrate / daily
   이미지:  Artifact Registry  (api · web · etl)
   외부 API: 열린국회(ASSEMBLY) · data.go.kr · 열린재정 · Gemini(요약)
```

| 컴포넌트 | GCP 리소스 | 이름 | 공개 |
|---|---|---|---|
| 프론트 (Next.js) | Cloud Run 서비스 | `pyosim-web` | 공개 |
| 백엔드 (FastAPI) | Cloud Run 서비스 | `pyosim-api` | 공개 |
| ETL 배치 | Cloud Run Job + Scheduler | `pyosim-daily` | 비공개(OAuth) |
| 마이그레이션 | Cloud Run Job (1회성) | `pyosim-migrate` | 비공개 |
| DB | Cloud SQL (Postgres 16) | `pyosim-db` | 비공개 |
| 비밀값 | Secret Manager | `database-url` 등 | — |
| 이미지 | Artifact Registry | `pyosim` repo | — |

## 2. 핵심 주의점 (운영과 로컬의 차이)

1. **AI 요약 provider 전환** — 로컬 기본값은 `ollama`(컨테이너에 없음).
   운영은 `SUMMARY_PROVIDER=gemini` + `GEMINI_API_KEY`(Secret). deploy.sh가 이미 그렇게 설정.
2. **프론트 API 주소는 빌드 시점에 박힘** — `NEXT_PUBLIC_API_BASE`는 Next 번들에 고정되므로
   **백엔드를 먼저 배포→URL 확정→프론트 빌드** 순서 필수. deploy.sh `frontend` 단계가 자동 처리.
3. **CORS** — 백엔드 `CORS_ORIGINS`에 프론트 URL을 넣어야 함. deploy.sh `frontend` 끝에서 자동 갱신.
4. **ETL 이미지 = backend + etl** — `etl/jobs/db.py`가 `../backend`(app.models)를 import.
   그래서 ETL Dockerfile은 **레포 루트**를 빌드 컨텍스트로 쓴다(`-f etl/Dockerfile .`).
5. **DB 연결** — Cloud Run은 Cloud SQL을 유닉스 소켓으로 붙음:
   `postgresql+psycopg://USER:PW@/pyosim?host=/cloudsql/PROJECT:REGION:pyosim-db`

## 3. 사전 준비

- `gcloud` CLI 설치 + `gcloud auth login`
- Docker 설치(로컬 빌드 대안. 기본은 Cloud Build 사용이라 없어도 됨)
- GCP 프로젝트 + 결제 계정 연결

```bash
export PROJECT_ID=<your-project-id>
export REGION=asia-northeast3      # 서울 (기본값)
```

## 4. 배포 순서

> Windows라면 Git Bash 또는 WSL에서 실행(스크립트가 bash).

```bash
bash infra/deploy.sh setup      # 1) API 활성화 + Artifact Registry
bash infra/deploy.sh db         # 2) Cloud SQL 인스턴스/DB/유저 (수 분)
bash infra/deploy.sh secrets    # 3) Secret Manager (키 대화형 입력)
bash infra/deploy.sh backend    # 4) 백엔드 배포 → URL 출력
bash infra/deploy.sh migrate    # 5) alembic upgrade head + 정당 시드
bash infra/deploy.sh frontend   # 6) 프론트 배포(백엔드 URL 자동 주입) + CORS 갱신
bash infra/deploy.sh etl        # 7) ETL Job + 매일 04:10 KST 스케줄
```

`secrets`까지 끝났으면 4~7을 한 번에: `bash infra/deploy.sh all`

### Secret 접근 권한 (한 번만)

Cloud Run 기본 서비스계정에 Secret 읽기 권한 부여:

```bash
PNUM=$(gcloud projects describe "$PROJECT_ID" --format='value(projectNumber)')
gcloud projects add-iam-policy-binding "$PROJECT_ID" \
  --member "serviceAccount:${PNUM}-compute@developer.gserviceaccount.com" \
  --role roles/secretmanager.secretAccessor
```

## 5. 배포 후 확인

```bash
# 백엔드 헬스
curl https://<api-url>/health
# 파이프라인 신선도(자동 갱신 모니터링)
curl https://<api-url>/api/health/pipeline
# ETL 즉시 1회 실행 테스트
gcloud run jobs execute pyosim-daily --region "$REGION" --wait
```

- 프론트: 출력된 `pyosim-web` URL 접속.
- 로그: Cloud Console → Cloud Run → 서비스/Job → 로그.

## 6. 재배포 (코드 변경 시)

```bash
bash infra/deploy.sh backend    # 백엔드만
bash infra/deploy.sh frontend   # 프론트만 (백엔드 URL 자동 반영)
bash infra/deploy.sh etl        # ETL 로직 변경 시
```

스키마 변경(새 alembic 리비전) 후엔 `bash infra/deploy.sh migrate` 한 번 더.

## 7. 비용 메모

- Cloud Run: 요청 없을 때 0으로 스케일 → 저트래픽이면 사실상 무료~소액.
- Cloud SQL `db-f1-micro`: 상시 과금(월 ~$10 안팎). 가장 큰 고정비.
  비용 더 줄이려면 인스턴스 중지 또는 외부 무료 Postgres(Neon/Supabase) + `DATABASE_URL`만 교체도 가능.
- Artifact Registry/Secret Manager: 미미.
